## 中文 markdown 字数检查

**Decision**：本项目（含其他中文为主的 markdown 项目）的字数验收一律用 `wc -m`（字符数）而非 `wc -w`（单词数），并按"1 中文字 ≈ 2-3 字符"换算上下界。

**Why**：本次设计文档约定 pedagogy/*.md "200-500 字"，蓝队用 `wc -w` 自检（输出都在 200-225 范围），自认达标。但红队脚本用 `wc -m` 字符数检查，发现 5 份 pedagogy 都在 2386-3058 字符，远超 500 字的合理换算（约 ≤2000 字符）。
- `wc -w` 把连续中文段算作 1 个 word（按空白分隔），无法反映真实字数
- `wc -m` 数 unicode 字符数，1 中文字 = 1 字符，比 wc -w 准确
- 但 `wc -m` 也会数 markdown 标记（##、空格、换行），所以"500 中文字"实际对应约 800-2000 字符（视段落标题密度而定）

最终蓝队压缩后实际 938-1259 字符，对应约 350-450 中文字，符合"200-500 字"的设计意图。

**How to apply**：
- 设计文档约定字数限制时，**同时**写"中文字数 X-Y"和"对应字符数约 P-Q（用 wc -m 测）"，避免后续验证歧义
- 写自检脚本时统一用 `wc -m` 而非 `wc -w`
- 字符数上下界按经验换算：纯中文段落 1 字 ≈ 1 字符；中英混排 markdown 1 字 ≈ 2-4 字符（取决于英文术语和 markdown 标记密度）
